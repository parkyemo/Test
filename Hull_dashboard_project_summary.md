# Hull 실적 대시보드 프로젝트 - 진행 정리 (Cowork 이관용)

> 회사 DB 직접 접근 불가로 개인 환경에서 진행. 실제 DB 헤더 구조 기반으로 더미데이터를 생성하여 Streamlit 대시보드를 연습 제작하는 프로젝트.

## 전체 진행 단계
1. ✅ DB 테이블 구조(헤더) 파악 및 용어 정의
2. ✅ 더미데이터 생성 (규칙 확정 + 스크립트 작성/검증 완료)
3. ⬜ Streamlit 대시보드 설계/제작 (다음 단계, Cowork에서 진행 예정)
4. ⬜ (추후) 실제 회사 DB 연결 시 참고 활용

---

## 1단계. 테이블 구조 및 용어 정의

### 테이블 3종
1. **Hull_erec_baseMH** - 블록/모듈 단위 작업 실적(물량, 공수)
2. **Hull_info** - 호선(프로젝트) 마스터 정보
3. **R99_rate** - 선종/stage별 실행공수 산출 factor

### Hull_erec_baseMH 헤더
```
프로젝트, 모블록번호, 블록번호, stage, 기능, a2, 모듈,
취부장_total, 취부장_butt, 취부장_fillet,
용접장_total, 용접장_butt, 용접장_fillet,
작업량, 작업량_단위,
심출_계량, 취부_계량, 용접_계량, 사상_계량, 기타_계량, total_계량,
심출_기준, 취부_기준, 용접_기준, 사상_기준, 기타_기준, total_기준,
원본파일명
```

**용어 정의**
- **프로젝트**: 호선 고유번호, SNXXXX 형식
- **모블록번호**: 하위 블록번호 여러 개가 합쳐진 상위 개념. 패턴: 영문1+숫자2+영문2 (예: B11UC)
- **블록번호**: 모블록의 하위 자블록. 패턴: 모블록과 동일 접두(영문1+숫자2) + 숫자1+영문1 (예: B110P). **모블록 없이 블록번호가 단독 존재하는 경우는 없음.**
  - stage 30/40 행: 블록번호 = 실제 자블록 코드
  - stage 50/60/70 행: 블록번호 = 모블록번호와 동일 값 (모블록 단위 집계 성격)
- **stage**: 작업 프로세스 단계. 30, 40, 50, 51, 52, 60, 70 (단, Hull_erec_baseMH 더미데이터에서는 자블록=30/40, 모블록레벨=50/60/70만 사용. 51/52는 R99_rate 테이블에만 존재)
- **기능**: 더미데이터 시 'D' 고정
- **a2**: 구분자. 형식 = `H + 블록번호(5자리) + 공정(SF0 고정) + stage(2자리) + 0` = 총 12자리
  - 예: 블록번호 B11UC, stage 40 → `HB11UCSF0400`
  - 실제 DB의 모블록번호/블록번호 생성 패턴 자체는 회사 자료라 비공개. 더미데이터는 위 규칙으로 생성.
- **모듈**: 독립적인 영문 문자열(5~7자). 하나의 (블록번호-stage) 조합 하위에 동일한 모듈 세트(4~6개)가 대부분 반복 사용됨
- **취부장/용접장**: welding length(길이 단위). total/butt/fillet 구성, total = butt + fillet
- **계량/기준**: 공수(Job Man Hour). 기준 = 계량 × factor (factor는 계량이 항상 기준보다 크도록 설정, 선종별 고정값)
  - total_계량 = 심출_계량 + 취부_계량 + 용접_계량 + 사상_계량 + 기타_계량 (기준도 동일 구조)
- **작업량 / 작업량_단위**: 대시보드에서 사용하지 않는 항목 (더미데이터는 임의값)
- **원본파일명**: 더미데이터에서는 참고용 임의 문자열

### Hull_info 헤더
```
도크/SKID, 프로젝트, 시리즈, 선주, 선종, 선형, 모선
```
- **선종**: LNG, CONT, SHTL, LPG 4종 (대시보드에서 핵심적으로 사용 - factor/R99율 조인 키)
- 도크/SKID, 시리즈, 선주, 선형, 모선: 이번 대시보드 연습에서는 **사용하지 않음** (단, 실제 DB 연결 시에는 사용할 수도 있어 구조상 유지)

### R99_rate 헤더
```
선종, stage, R99율
```
- R99율은 계량/기준 factor와는 **완전히 다른 개념**의 factor
- 대시보드에서 **실행공수 = 기준공수 × (1 + R99율)** 산출 시 사용 예정

## 2단계. 더미데이터 생성 규칙 (확정)

### 규모
- 프로젝트(호선): 10개, 선종(LNG/CONT/SHTL/LPG) 랜덤 배정
- 모블록번호: 프로젝트당 5개
- 블록번호: 모블록당 8개 (자블록)
- stage 구조:
  - 자블록 8개 각각 → stage 30, 40 두 개씩 보유
  - 모블록 레벨(블록번호=모블록번호) → stage 50, 60, 70 보유
- 모듈: (블록번호-stage) 조합 하위에 4~6개, 모블록 단위로 생성한 동일 모듈풀에서 반복 샘플링

### 수치 생성 규칙
| 항목 | 규칙 |
|---|---|
| 취부장/용접장 (butt/fillet) | 각각 50~450m 랜덤, total = butt + fillet |
| total_계량 | 100~1000 랜덤 |
| 계량공수 항목별 비율 | 심출 11%, 취부 22%, 용접 54%, 사상 10%, 기타 3% (합 100%) |
| 계량→기준 factor (선종별 고정) | LNG 0.8, CONT 0.7, SHTL 0.85, LPG 0.85 |
| 작업량/작업량_단위 | 랜덤 또는 공백 (대시보드 미사용) |
| 도크/SKID, 시리즈, 선주, 선형, 모선 | 랜덤 또는 공백 (대시보드 미사용) |

### R99_rate 값 (지정값, 그 외 stage는 0)
| 선종 | 40 | 50 | 51 | 52 | 60 |
|---|---|---|---|---|---|
| LNG | 0.221 | 0.312 | 0.312 | 0.169 | 0.378 |
| CONT | 0.225 | 0.349 | 0.349 | 0.280 | 0.147 |
| SHTL | 0.228 | 0.444 | 0.444 | 0.000 | 0.370 |
| LPG | 0.145 | 0.145 | 0.145 | 0.145 | 0.145 |

(stage 30, 70은 모든 선종 0)

### 산출물 (생성 완료, 검증 완료)
- `Hull_erec_baseMH.csv` (4,402행 x 28열)
- `Hull_info.csv` (10행 x 7열)
- `R99_rate.csv` (28행 x 3열)
- `generate_dummy_data.py` (생성 스크립트, seed=42로 재현 가능)

**검증 완료 항목**
- stage별 행 분포 정상 (자블록=30/40, 모블록레벨=50/60/70)
- total_계량 = 5개 항목 합 (오차 반올림 수준)
- total_기준 = 5개 항목 합 (오차 반올림 수준)
- a2 패턴 정상 생성 (예: `HR877RSF0300`)
- R99_rate 매트릭스 지정값/미지정값(0) 정상 반영
