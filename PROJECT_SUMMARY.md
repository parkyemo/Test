## 🎯 구현된 분석 기능

### 필터링 (동적)
```
✅ 프로젝트 선택 (단일/복수)
✅ 선종 선택 (LNG/CONT/SHTL/LPG)
✅ Stage 선택 (30/40/50/51/52/60/70)
✅ 필터 결과 실시간 동기화
```

### KPI 지표 (5개)
```
1️⃣ 계량 총합 (∑ total_계량)
2️⃣ 기준 총합 (∑ total_기준)
3️⃣ 실행공수 총합 (∑ total_실행)
4️⃣ 기준 대비 증가율 (%)
5️⃣ 평균 R99율 (%)
```

### 시각화 차트
```
📈 Tab 1: 그룹 바 차트 (프로젝트별)
🥧 Tab 2: 도넛 차트 (선종별 비율)
📊 Tab 3: 라인 차트 (Stage별 추이)
🔧 추가: 수평 바 차트 (공정별 분포)
```

### 데이터 테이블
```
📋 상세 데이터 조회 (필터된 행)
💾 CSV 다운로드
🔄 정렬 & 스크롤
```

## 🔧 핵심 계산 로직

### 1. 데이터 조인
```
Hull_erec_baseMH (4,402행)
         ↓ [프로젝트 조인]
    Hull_info (10행)
         ↓ [선종+stage 조인]
    R99_rate (28행)
         ↓
  통합 데이터프레임
```

### 2. 실행공수 산출
```
기준공수 = 계량공수 × factor
  factor: LNG 0.8, CONT 0.7, SHTL 0.85, LPG 0.85

실행공수 = 기준공수 × (1 + R99율)
  예) 기준 100 × (1 + 0.221) = 122.1 (LNG stage 40)
```

### 3. 5개 공정별 분해
```
total_실행 = 심출_실행 + 취부_실행 + 용접_실행 + 사상_실행 + 기타_실행
  - 계량 비율: 심출 11%, 취부 22%, 용접 54%, 사상 10%, 기타 3%
```

## 💡 주요 설계 결정

### 1. CSV → DB 전환 계획
- **현재**: CSV 기반 (프로토타이핑)
- **향후**: 실제 DB 기반 (운영)
- **전환 방식**: 코드 수정 최소화 (함수 캡슐화)

### 2. 캐싱 전략
- **@st.cache_data**: 데이터 쿼리 결과 (ttl=3600초)
- **@st.cache_resource**: DB 연결 객체 (수명주기 길음)

### 3. 실행공수 계산
- **선택 이유**: 기준공수에 R99율 반영 → 현실적 공수 예측
- **적용 대상**: 모든 공정(심출/취부/용접/사상/기타)

### 4. Streamlit 선택
- **장점**: 빠른 프로토타이핑, 배포 간편, Python 기반
- **단점**: 커스터마이징 한계 → 향후 대시보드 업그레이드 시 고려
