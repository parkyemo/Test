# Hull 실적 대시보드 — 작업 재개용 컨텍스트

> 이 문서는 새로운 채팅 세션(회사 chat 환경 등)에서 AI에게 그대로 붙여넣어 작업 맥락을 전달하기 위한 브리핑 문서입니다.
> `hull_dashboard.py`, `requirements.txt`와 함께 사용하세요.

---

## 1. 프로젝트 한 줄 요약

조선소 Hull(선체) 블록 단위 작업 실적(공수)을 프로젝트/선종/STAGE별로 조회·분석하는 **Streamlit 대시보드**. 회사 DB에 직접 접근할 수 없는 환경에서 만들었기 때문에, 지금은 실제 DB 테이블과 **헤더(컬럼명)를 동일하게 맞춘 더미데이터 CSV 3개**로 동작한다.

## 2. 데이터 구조 (= 실제 DB 스키마와 동일한 컬럼명)

| 파일 | 역할 | 핵심 컬럼 |
|---|---|---|
| `Hull_erec_baseMH.csv` | 블록/모듈 단위 작업 실적 (원장) | 프로젝트, 모블록번호, 블록번호, stage, 모듈, 용접장_total, 심출_계량~기타_계량, 심출_기준~기타_기준, total_계량, total_기준 |
| `Hull_info.csv` | 호선(프로젝트) 마스터 | 프로젝트, 선종, 선형, 선주, 시리즈 |
| `R99_rate.csv` | 선종×STAGE별 실행공수 증가율 | 선종, stage, R99율 |

세 파일을 `프로젝트`, `선종+stage` 기준으로 merge해서 통합 데이터프레임 `df`를 만들고, 이후 모든 로직(필터/KPI/차트)은 이 `df`의 **컬럼명만 참조**한다.

## 3. 실제 DB 연동 시 해야 할 일

- `hull_dashboard.py`의 `load_data()` 함수 하나만 CSV 읽기 → DB 쿼리(`pd.read_sql` 등)로 교체하면 됨. **쿼리 결과 컬럼명을 CSV 헤더와 동일하게** 맞추는 게 핵심.
- 예외 1곳: `info_df_lookup` 변수(파일 내 `Hull_info.csv`를 `load_data()`와 별개로 한 번 더 읽는 부분, 프로젝트 정보 툴팁용) — 이것도 같이 DB 쿼리로 바꿔야 함.
- `@st.cache_data`가 걸려있어 캐싱됨. 실시간 반영이 필요하면 `ttl=` 파라미터 추가 고려.

## 4. 핵심 비즈니스 로직 — 절대 규칙

### 실행공수
```
실행공수 = 기준공수 × (1 + R99율)
```
R99율은 (선종, stage) 조합 하나당 값 하나이며, 한 행(모듈) 안의 5개 공정(심출/취부/용접/사상/기타)에 **동일하게** 적용된다.

### 원단위 (unit-rate) — ⚠️ 이번 작업의 핵심 신규 개념
```
원단위 = 공수값 ÷ 용접장_total  (소수점 4자리)
```
**여러 행을 하나의 원단위로 합칠 때는 절대로 개별 원단위 값을 평균 내면 안 된다.** 반드시 아래처럼 분자·분모를 각각 SUM한 뒤 마지막에 한 번만 나눌 것 (가중합계비 방식):

```python
def calc_wondanwi(numerator_sum, denom_sum):
    if denom_sum == 0:
        return 0.0
    return round(numerator_sum / denom_sum, 4)
```

- 그룹(모듈→블록→모블록→프로젝트→선종→전체) 어느 단위든 이 원칙은 동일.
- 용접장_total이 0인 행은 원단위 0으로 표시하되, 그 행의 분자값은 상위 집계 합산에는 정상 포함.
- "평균"이라는 표현이 나와도 **공수(절대값)는 총합÷개수**, **원단위(비율)는 위 방식**으로 구분해서 처리.
- (검증 사례: 프로젝트별 원단위를 단순평균 vs 가중합계비로 계산하면 실제로 다른 숫자가 나옴 — CONT 선종 기준 0.8865 vs 0.8844)

## 5. 현재 구현된 기능 요약

- **사이드바 필터 5개** (프로젝트/선종/모블록번호/블록번호/Stage): 전부 상호 교차 연동됨 (어느 것을 먼저 선택해도 나머지가 즉시 좁혀짐). 초기화 버튼은 위젯 key에 카운터를 붙여 완전 리셋되도록 구현.
- **주요지표(KPI) 6개 한 줄**: 계량공수합/기준공수합/실행공수합/계량원단위/기준원단위/실행원단위
- **탭 4개** (색상 구분: 파랑/초록/주황/회색):
  - 프로젝트별: 프로젝트별 원단위 그룹바 차트 + 테이블
  - 선종별: 사이드바 필터 무시, 전체 DB 기준 — 선종별(LNG/CONT/LPG) 호선별 실행공수+실행원단위 콤보차트 3개(세로) + 우측 선종별 호선당 평균 요약(가로 테이블)
  - STAGE별: STAGE별 원단위 추이 라인차트 + 테이블(기준/실행원단위를 계량원단위 대비 %로 병기)
  - 상세 데이터: 필터된 원본 행 테이블 + CSV 다운로드
- **공정별 공수 분석**: 심출/취부/용접/사상/기타 비율을 계량/기준/실행 3개 도넛 파이차트로 (가운데 라벨+총합 표시)
- **프로젝트 비교 모드**: 프로젝트 A/B를 좌우로 비교. 사이드바의 **선종·모블록번호 필터만** 이 모드에 반응(블록/Stage 필터는 무시). 선종 선택 시 A/B 드롭다운이 해당 선종 프로젝트로 좁혀짐. 블록+Stage 단위 원단위 편차(최대-최소) 표시, 모블록이 한쪽에만 있으면 "해당 모블록번호는 없습니다" 안내.

## 6. 알려진 더미데이터의 한계 (실제 DB에서는 다를 수 있음)

- 블록번호는 프로젝트 간 전혀 겹치지 않음 (실제 DB에서는 동일 선종이면 겹칠 수 있다고 함).
- 심출/취부/용접/사상/기타 공정 비율이 모든 행에서 정확히 11/22/54/10/3%로 고정 생성되어 있어, 계량/기준/실행 파이차트 비율이 사실상 동일하게 나옴 (R99율이 공정 구분 없이 행 단위로만 존재하기 때문). 실제 DB에 공정별 R99율이 따로 있다면 달라질 수 있음.
