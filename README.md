### 진행 현황
- ✅ **1단계**: DB 테이블 구조 및 용어 정의 완료
- ✅ **2단계**: 더미데이터 생성 완료 (`generate_dummy_data.py`)
- ✅ **3단계**: **Streamlit 대시보드 완성** (`hull_dashboard.py`)

## 🎯 핵심 기능

### 1️⃣ 실시간 필터링
- **프로젝트** 선택: 특정 호선(SN 시리즈) 필터링
- **선종** 선택: LNG, CONT, SHTL, LPG 중 선택
- **Stage** 선택: 30, 40, 50, 51, 52, 60, 70 공정 단계 선택
- 필터 결과 실시간 반영 → 상단 KPI 및 모든 차트 동기화

### 2️⃣ 주요 지표 (KPI)
상단에 5개 지표를 한눈에 표시:
- **계량 총합**: ∑ total_계량
- **기준 총합**: ∑ total_기준
- **실행공수 총합**: ∑ total_실행
- **기준 대비 증가율**: (실행공수 ÷ 기준공수 - 1) × 100 %
- **평균 R99율**: mean(R99율) × 100 %

### 3️⃣ 탭별 분석 (4개 탭)

#### 📈 Tab 1: 프로젝트별 분석
- 그룹 바 차트: 프로젝트별 계량/기준/실행공수 3중 비교
- 테이블: 프로젝트별 집계 데이터
- 용도: 호선(프로젝트)별 공수 추이 추적, 진척률 비교

#### 🚢 Tab 2: 선종별 분석
- 도넛 차트: 선종별 실행공수 비율 시각화
- 테이블: 선종별 계량/기준/실행공수 + 평균 R99율
- 용도: 선형 타입별 공수 분포, 난이도 비교

#### 📍 Tab 3: Stage별 분석
- 라인 차트: 30→40→50→60→70 공정 단계별 추이
- 테이블: Stage별 집계 데이터
- 용도: 공정 병목 지점 파악, 단계별 부하 분석

#### 📋 Tab 4: 상세 데이터
- 전체 필터된 행 조회 (프로젝트, 모블록, 블록, stage, 선종, 모듈, 공수 등)
- CSV 다운로드: 필터된 데이터를 로컬 저장 가능

### 4️⃣ 추가 분석: 공정별 공수 분포
- 수평 바 차트 좌측: 심출, 취부, 용접, 사상, 기타 **계량** 분포
- 수평 바 차트 우측: 위 5가지 공정의 **실행공수** 분포
- 용도: 공정 유형별 작업 강도 파악

## 📊 계산 로직

### 1. 데이터 조인 흐름
```
Hull_erec_baseMH (실적 데이터)
           ↓ [프로젝트 조인]
        Hull_info (선종 매핑)
           ↓ [선종+stage 조인]
        R99_rate (R99율)
           ↓
    통합 데이터프레임
```

### 2. 공수 계산
```
기준공수 = 계량공수 × factor (선종별 고정값)
  - LNG:  0.80
  - CONT: 0.70
  - SHTL: 0.85
  - LPG:  0.85

실행공수 = 기준공수 × (1 + R99율)
  예) LNG, stage 40, 기준공수 100
      → 실행공수 = 100 × (1 + 0.221) = 122.1

공정별 분해:
  - total_기준 = 심출_기준 + 취부_기준 + 용접_기준 + 사상_기준 + 기타_기준
  - total_실행 = 심출_실행 + 취부_실행 + 용접_실행 + 사상_실행 + 기타_실행
```

### 3. R99율 (선종별 지정값)
| 선종 | Stage 40 | 50 | 51 | 52 | 60 |
|------|---------|-----|-----|-----|-----|
| LNG  | 0.221  | 0.312 | 0.312 | 0.169 | 0.378 |
| CONT | 0.225  | 0.349 | 0.349 | 0.280 | 0.147 |
| SHTL | 0.228  | 0.444 | 0.444 | 0.000 | 0.370 |
| LPG  | 0.145  | 0.145 | 0.145 | 0.145 | 0.145 |

*(Stage 30, 70은 모든 선종 R99율 = 0)*

## 📝 주요 설계 결정

### 1. 왜 Streamlit인가?
- ✅ 프로토타이핑 빠름 (Python 코드만으로 UI 자동 생성)
- ✅ 데이터 시각화에 최적화 (Plotly 통합)
- ✅ 배포 간편 (Streamlit Cloud)
- ✅ 비용 효율적

### 2. 왜 Plotly를 선택했나?
- ✅ 인터랙티브 차트 (마우스 호버, 줌/팬)
- ✅ 자동 범례 필터링
- ✅ 다양한 차트 타입 (막대, 선, 파이 등)

### 3. 데이터 캐싱 (@st.cache_data)
- CSV 파일을 매 세션마다 새로 로드하는 대신 캐싱
- 필터 상호작용 시 빠른 응답
- 실제 DB 연결 시에는 쿼리 캐싱으로 전환
